# redhat_satellite

Collects OpenSCAP reports from Red Hat Satellite for the hosts targeted by the current inventory, packages the results into a password-protected ZIP, and sends the archive by email.

## How it works

The role is currently driven by one entrypoint:

| Variable | Module | Description |
|---|---|---|
| `REDHAT_SATELLITE_OPENSCAP_REPORTING: true` | `tasks/reporting/` | Fetches OpenSCAP ARF reports from Satellite, exports them as HTML, encrypts the archive through PMP, and sends it by mail |

The role expects:
- `client_id` to be defined by the client inventory, typically in `group_vars/all`
- `common` to be loaded first so that Satellite, PMP, and mail variables are available
- execution from `localhost` for all file generation, encryption, and mail delivery steps

The OpenSCAP scope is built from:
- the hosts present in the current play execution scope
- intersected with the hosts found in the Satellite location named after `client_id`

---

## OpenSCAP reporting

The role:
- lists hosts in the Satellite location `client_id`
- lists ARF reports for each selected host
- keeps the latest reports according to `__satellite_reports_per_host`
- downloads the full HTML rendering of the report from Satellite
- optionally downloads the raw ARF `.xml.bz2`
- builds a ZIP archive encrypted with the PMP `Comm Cli` password
- sends the result by email

### Output format

By default, the archive contains only the Satellite-generated HTML report.

If `__satellite_attach_raw_arf: true`, the archive also includes the raw `.xml.bz2` ARF payload.

### Limiting the execution scope

The playbook uses a dynamic `execution_group` so that a limiter can be applied without processing the whole client inventory.

Examples:

```bash
# Full client inventory
ansible-playbook playbooks/managed_services/redhat_satellite/openscap_reporting.yaml \
  -i clients/XXXX/hosts.ini

# Single host from the client inventory
ansible-playbook playbooks/managed_services/redhat_satellite/openscap_reporting.yaml \
  -i clients/XXXX/hosts.ini \
  -e "limiter=sv-xxxxlvuxx.rh.xxxx.local"
```

## Variables reference

| Variable | Default | Description |
|---|---|---|
| `__use_internal_communication_password` | `false` | Forces the `2000` CommCli password for ZIP encryption |
| `__satellite_target_hosts_override` | `[]` | Optional explicit list of hosts to process instead of the current play scope |
| `__satellite_output_zip_file` | `""` | Override for the generated ZIP filename |
| `__satellite_reports_per_host` | `1` | Number of latest reports to keep per host. `0` means all reports |
| `__satellite_prefer_satellite_html` | `true` | Prefer the full HTML rendering generated by Satellite |
| `__satellite_attach_raw_arf` | `false` | Include the raw `.xml.bz2` ARF report in the archive |
| `__satellite_report_download_timeout` | `120` | Timeout in seconds for Satellite report downloads |

---

## Notes

- The role relies on `__satellite_https_url`, `__satellite_svc_user`, and `__satellite_svc_token` from the `common` role.
- The HTML report endpoint currently used is under `/api/compliance/arf_reports/<id>/download_html`.
- Archive encryption reuses the PMP logic already used elsewhere in the repository.
- If PMP is unreachable, the fallback password logic from the existing utility tasks is used.
